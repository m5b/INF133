Two Approaches to Mobile Development

Native
  • An app designed to work on a specific piece of hardware
  • Usually built with tools created by the hardware/platform manufacturer
  • e.g., android studio (Android) and Xcode (iOS)
Web
  • Mobile-first responsive web app viewed through browser
  • Hybrid-native built with web tools, but rendered in native wrapper
  • Prgoressive Web App or PWA, 'install' to a device (but really just web app)

Trade-offs?

Native
  • Platform specific codebase
  • Smaller dev pool
  • Must maintain multiple codebase
  • Faster, 'closer to the metal'
Web
  • Universal codebase
  • Large dev pool
  • Use a common codebase
  • Slower, limited access to device, non-native look
